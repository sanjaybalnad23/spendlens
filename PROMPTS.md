# PROMPTS

## AI Audit Summary Prompt

This is the prompt currently used for generating personalized audit summaries.

The audit math and recommendations themselves are NOT AI-generated. The AI is only used to convert the structured audit result into a short human-readable summary.

That decision was intentional because deterministic rules felt much more reliable for financial recommendations than letting an LLM hallucinate savings logic.

---

# System Prompt

```txt id="m1q8x2"
You are an AI financial optimization assistant.

Your job is to explain AI SaaS audit results in concise,
business-friendly language.

Keep summaries:
- practical
- short
- actionable
- non-technical
```

---

# User Prompt

```txt id="p1q8x2"
Audit Input:

${JSON.stringify(params, null, 2)}

Audit Result:

${JSON.stringify(result, null, 2)}

Generate a concise AI audit summary.
```

---

# Why I Wrote It This Way

Initially I tried much larger prompts with more instructions and formatting rules.

The outputs actually became worse and more robotic.

Keeping the prompt simple produced:

- shorter summaries
- less hallucination
- more readable outputs
- less over-explaining

I also intentionally avoided asking the AI to:

- calculate savings
- decide recommendations
- choose plans
- replace the rule engine

The AI only explains the result already generated by deterministic business logic.

---

# What Didn’t Work

Some things I tried that produced bad results:

- Asking the AI to fully generate recommendations
- Letting the model decide savings opportunities directly
- Very long “consultant style” prompts
- Overly formatted prompts with too many constraints

Problems I saw:

- fake confidence
- unrealistic savings claims
- repetitive summaries
- weird markdown formatting
- recommendations contradicting the actual engine output

This is why the final system uses:

- rule-based audit logic
- AI only for summary generation

---

# Fallback Behavior

If AI summary generation fails, the app can still function because the core audit engine is fully deterministic.

A fallback templated summary can be returned instead of blocking audit generation completely.
